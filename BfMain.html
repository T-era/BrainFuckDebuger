About expanded syntax

In this tool, script parser have function "Macro", expanded from standerd B.F.

What's "Macro"?

Additional command, which can use to replace for some mass of commands (in this tool internal).

Push button "+", in right edge of screen. Added new tab (which you can name), you can call it as name in main script (or in another "Macro").
Macro's name must be one character.(or it can't be called by anywhere.)

How to edit "Macro" contents?

Push button values their name. Once, view contents. Second, open text-area and you can edit it.

Is it function ?

No. Sometime, it works as like function. But isn't same.
B.F. parser try to expand all macros at the first. If macro was recursional, parser would never complete it.

Any other limitations ?

A Macro must be closed in syntax. Must matche Loop start "[" and loop end "]".

About debug

How to execute in debugger ?

To use debugger...

  • Button "Run" for evaluate all script.
  • Button "A step" for evaluate a step.
  • Button "To break point" for execute script until break-point or end of script.

How to set break-point ?

Click on a command, on view-mode.(Not edit-mode.)
